Hexo 博客主题之 Matery 使用

Word Count:   5.3k

hexo-theme-matery 是一个采用 Material Design 和响应式设计的 Hexo 博客主题。本文可以看做是 Matery 主题的 README 文档，以供初次使用 Matery 主题的 Hexoer 们，Matery 主题效果演示 见我的个人博客。

1. 特性支持

Matery 主题提供了丰富，灵活，美观的特性支持，你可以根据根据个人需要定制化的选择相应的特性支持：

  • 简单漂亮，文章内容美观易读
  • Material Design 设计
  • 响应式设计，博客在桌面端、平板、手机等设备上均能很好的展现
  • 首页轮播文章及每天动态切换 Banner 图片
  • 瀑布流式的博客文章列表（文章无特色图片时会有 24 张漂亮的图片代替）
  • 时间轴式的归档页
  • 词云的标签页和雷达图的分类页
  • 丰富的关于我页面（包括关于我、文章统计图、我的项目、我的技能、相册等）
  • 可自定义的数据的友情链接页面
  • 支持文章置顶和文章打赏
  • 支持 MathJax
  • TOC 目录
  • 可设置复制文章内容时追加版权信息
  • 可设置阅读文章时做密码验证
  • Gitalk、Gitment、Valine 和 Disqus 评论模块（推荐使用 Gitalk）
  • 集成了不蒜子统计、谷歌分析（Google Analytics）和文章字数统计等功能
  • 支持在首页的音乐播放和视频播放功能
  • 支持emoji表情，用markdown emoji语法书写直接生成对应的能跳跃的表情。
  • 支持 DaoVoice、Tidio 在线聊天功能。

Update Time：2020/02/20，详细特性支持配置方法见后文 Matery 主题配置部分。

2. Matery 下载以及启用

我们知道，Hexo 站点目录下的 themes 目录专门用来存放 Hexo 主题。通过查看 themes 目录我们发现，Hexo 安装后默认只有一个 landscape 主题（如果你安装有其它主题的话，也可以在这里找到）。

Matery 安装启用方法非常简单（事实上，所有 Hexo 主题的安装启用过程都类似）：

2.1 主题下载

点击 这里 下载 master 分支的最新稳定版的代码，解压缩后，将 hexo-theme-matery 的文件夹复制到你 Hexo 的 themes 文件夹中即可。

当然，你也可以进入到你的 themes 文件夹下使用 Git clone 命令来直接下载:

$ cd <站点目录>/themes
$ git clone https://github.com/blinkfox/hexo-theme-matery.git

2.2 主题启用

Matery 主题下载好之后，我们需要将当前主题切换为 Matery：

在 Hexo 根目录下的 _config.yml 配置文件中查找 theme 关键字，修改其值为 theme: hexo-theme-matery，具体内容如下：

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
#theme: landscape   # Default Theme
theme: hexo-theme-matery

2.3 主题验证

Matery 下载启用之后，我们需要进行一下本地验证（预览），以检验主题是否成功切换到 Matery？

注意本地预览之前，需要清除一下本地 Hexo 缓存：

$ hexo clean
INFO  Deleted database.
INFO  Deleted public folder.

然后，启动 Hexo 本地服务进行预览（验证）：

$ hexo s
INFO  Start processing
INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

OK！浏览器中访问：http://localhost:4000，此时发现我们的个人博客主题已经发生变更。

成功启用了 Matery 主题之后，接下来我们就可以定制化的进行相关的特性支持配置了：

3. Matery 主题配置

开始之前，我们先给出两点必要的声明：

  1. 在 Hexo 中有两份主要的配置文件，其名称都是 _config.yml：一份位于站点根目录下，主要包含 Hexo 本身的配置；另一份位于主题目录下（由主题作者提供），主要用于配置主题相关的选项。为了后续描述方便，故将前者称为 站点配置文件， 后者称为主题配置文件；
  2. 以下所有终端（Git hash）执行的命令都在 Hexo 站点目录（根目录）下。

下文我们会看到上述 [1. 特性支持] 的配置方法（事实上，你还可以自己修改 Matery 主题样式，为避免博文篇幅过长，其它优化请关注后续系列文章）：

3.1 基本信息配置

基本信息包括：博客标题、副标题、作者、描述、语言等等。

打开 站点配置文件 ，查找 Site 关键字。配置内容如下：

# Site
title: Waldeinsamkeit           # 设置网站标题
subtitle: The Night Is Young    # 设置网站副标题
description:                    # 设置网站描述
keywords:
author: Waldeinsamkeit          # 设置作者（所有者）
language:                       # 设置语言
timezone:                       # 设置网站时区：Hexo 默认使用当前电脑的时区（默认即可）

关于语言设置：

我们需要在 <主题目录>/languages/{language}.yml 查看当前下载的 Next 主题所支持的语言类型，你可以看到一系列的 {language}.yml 文件。我的如下：

default.yml
zh-CN.yml

代表了当前主题所支持的语言类型有：default，zh-CN。如果你是中文用户，则建议修改 language 的值为 zh-CN（默认是英文）。

关于 基本信息配置 中的其他选项配置可参考 Hexo 官方站点配置文档。

3.2 站点 URL 配置

打开 站点配置文件 ，查找 URL 关键字。配置内容如下：

# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: http://yoursite.com                # 你网站的主 URL：http://xxx.github.io
root: /                                 # 网站根目录
#permalink: :year/:month/:day/:title/
permalink: :category/:title/            #文章的永久链接格式
permalink_defaults:

关于 Permalinks （永久链接），其支持的属性设定见链接地址。

关于 站点 URL 配置 中的其他选项配置可参考 Hexo 官方站点配置文档。

3.3 菜单配置

菜单包括：首页、归档、分类、标签、关于、留言板、友情链接等等。

3.3.1 添加/删除菜单项

默认情况下，菜单项均开启，我们可以通过个人需求删除某些菜单项（不需要哪个菜单项就将其注释掉）。另外菜单项配置内容包括：菜单导航的名称、路径 url 和图标 icon 等

1 –> 设置菜单项：

打开 主题配置文件 找到 Menu Settings 模块。如下：

# ---------------------------------------------------------------
# Menu Settings
# Main(The First) menu navigation url and icon
menu:
  HOME:
    url: /
    icon: fas fa-home
  Archives:
    url: /archives
    icon: fas fa-archive
  Tags:
    url: /tags
    icon: fas fa-tags
  Categories:
    url: /categories
    icon: fas fa-bookmark
  About Me:
    url: /about
    icon: fas fa-user-circle
  Contact:
    url: /contact
    icon: fas fa-comments
  # Friends:
  #   url: /friends
  #   icon: fas fa-address-book
# ---------------------------------------------------------------

菜单项设置格式说明，以 HOME 为例：

  • HOME 表示菜单项名称 “HOME”，修改后可直接显示在页面上；
  • url 表示 “HOME” 菜单项访问路径;
  • icon 表示”HOME” 菜单项对应图标

2 –> 设定菜单项的图标：

注意关于菜单项图标，可以去 Font Awesome 中选择修改（ Matery 主题所有的图标都来自 Font Awesome）。

3.3.2 初始化菜单项页面

添加菜单项后，其实是无法正常显示菜单项页面的（只是增加了一个菜单项按钮模块）。此时点击菜单项按钮可能显示（以 categories 为例）：

Cannot Get /categories/

下面我们以 categories 菜单项为例来看如何初始化其对应的 categories 菜单页面：

1 –> 初始化 categories页面，Git bash 站点目录下运行：

$ hexo new page categories
INFO  Created: F:\HexoBlogProject\source\categories\index.md

2 –> 初始化成功后，会在 <站点目录>/source 目录下出现一个 categories 目录（默认情况下只存在 _posts 目录）。

3 –> 编辑 <站点目录>/source/categories 中 index.md 文件，添加页面类型 type: "categories"。至少需要以下内容：

title: 分类
date: 2019-01-08 19:29:04
type: "categories"
layout: "categories"

注意其它菜单项（tags，about，contact，friends）页面可同理进行初始化。

此时，你再次尝试点击菜单项按钮发现已经可以正常跳转各个相应显示页面了!!!

3.3.3 设置二级菜单

如果你需要设置二级菜单，则可以在上述菜单导航的基础上如下操作（以 Medias 菜单项为例）：

  1. 在需要添加二级菜单的一级菜单下添加 children 关键字(如：Medias菜单下添加 children) ；
  2. 在 children下创建二级菜单的 名称 name，路径 url 和图标 icon；
  3. 注意每个二级菜单模块前要加 -；
  4. 注意缩进格式

示例内容如下：

Medias:
 icon: fas fa-list
 children:
   - name: Musics
     url: /musics
     icon: fas fa-music
   - name: Movies
     url: /movies
     icon: fas fa-film
   - name: Books
     url: /books
     icon: fas fa-book
   - name: Galleries
     url: /galleries
     icon: fas fa-image

3.4 设置代码高亮

由于 Hexo 自带的代码高亮主题显示不好看，所以主题中使用到了 hexo-prism-plugin 的 Hexo 插件来做代码高亮，安装命令如下：

$ npm i -S hexo-prism-plugin

然后，在 Hexo 站点配置文件中查找 highlight 关键字，并将其设置为：false（即关闭默认的代码高亮）。

并在下面新增 prism 插件相关的配置，主要配置如下：

highlight:
  enable: false
#......

# prism 配置内容：
prism_plugin:
  mode: 'preprocess'    # realtime/preprocess
  theme: 'tomorrow'
  line_number: false    # default false
  custom_css:

3.5 设置站内搜索

本主题中还使用到了 hexo-generator-search 的 Hexo 插件来做内容搜索，安装命令如下：

$ npm install hexo-generator-search --save

然后在 Hexo 站点配置文件中，新增以下的配置项：

search:
  path: search.xml
  field: post

3.6 中文链接转拼音

推荐安装！

如果你的文章名称是中文的，那么 Hexo 默认生成的永久链接也会有中文，这样不利于 SEO，且 gitment 评论对中文链接也不支持。

我们可以用 hexo-permalink-pinyin Hexo 插件使在生成文章时生成中文拼音的永久链接：

$ npm i hexo-permalink-pinyin --save

然后在 Hexo 站点配置文件中，新增以下的配置项：

permalink_pinyin:
  enable: true
  separator: '-' # default: '-'

补充：除了此插件外，hexo-abbrlink 插件也可以生成非中文的链接。

3.7 设置文章字数统计插件

推荐安装！

如果你想要在文章中显示文章字数、阅读时长等信息，可以安装 hexo-wordcount插件。

Git bash 站点目录下运行如下即可安装：

$ npm i --save hexo-wordcount

然后只需在 主题配置文件中查找 postInfo 关键字，将各个文章字数相关的配置激活即可：

postInfo:
  date: true
  update: false
  wordCount: false  # 设置文章字数统计为 true.
  totalCount: false # 设置站点文章总字数统计为 true.
  min2read: false   # 阅读时长.
  readCount: false  # 阅读次数.

3.8 设置 Emoji 表情支持

Matery 主题新增了对emoji表情的支持，使用到了 hexo-filter-github-emojis 的 Hexo 插件来支持 emoji 表情的生成。使用时会将把对应的markdown emoji语法[::]转变成会跳跃的 emoji表情（例如:smile:，显示效果为：:smile:）

Git bash 站点目录下运行如下即可安装：

$ npm install hexo-filter-github-emojis --save

然后在 Hexo 站点配置文件中，新增以下的配置项：

githubEmojis:
  enable: true
  className: github-emoji
  inject: true
  styles:
  customEmojis:

3.9 设置 RSS 订阅支持

本主题中还使用到了 hexo-generator-feed 的 Hexo 插件来做 RSS，Git bash 站点目录下运行如下即可安装：

$ npm install hexo-generator-feed --save

在 Hexo 站点配置文件中，新增以下的配置项：

feed:
  type: atom
  path: atom.xml
  limit: 20
  hub:
  content:
  content_limit: 140
  content_limit_delim: ' '
  order_by: -date

执行 hexo clean && hexo g 重新生成博客文件，然后在 public 文件夹中即可看到 atom.xml 文件，说明你已经安装成功了。

3.10 设置在线聊天功能

在线聊天功能二选其一：

3.10.1 DaoVoice 在线聊天功能

前往 DaoVoice 官网注册并且获取 app_id，并将 app_id 填入 主题配置文件 中。

3.10.2 Tidio 在线聊天功能

前往 Tidio 官网注册并且获取 Public Key，并将 Public Key 填入主题配置文件 中。

3.11 设置站点页脚

页脚信息可能需要做定制化修改，而且它不便于做成配置信息，所以可能需要你根据个人需要去修改和加工。

修改的地方在主题文件的 /layout/_partial/footer.ejs 文件中，包括站点、使用的主题、访问量等。

3.12 设置社交链接

在 主题配置文件 中，默认支持 QQ、GitHub 和邮箱等社交链接的配置。

你可以在主题文件的 /layout/_partial/social-link.ejs 文件中，新增、修改你需要的社交链接地址，增加链接可参考如下代码：

<% if (theme.socialLink.github) { %>
    <a href="<%= theme.socialLink.github %>" class="tooltipped" target="_blank" data-tooltip="访问我的GitHub" data-position="top" data-delay="50">
        <i class="fab fa-github"></i>
    </a>
<% } %>

其中，社交图标（如：fa-github）你可以在 Font Awesome 中搜索找到。以下是常用社交图标的标识，供你参考：

  • Facebook: fab fa-facebook
  • Twitter: fab fa-twitter
  • Google-plus: fab fa-google-plus
  • Linkedin: fab fa-linkedin
  • Tumblr: fab fa-tumblr
  • Medium: fab fa-medium
  • Slack: fab fa-slack
  • Sina Weibo: fab fa-weibo
  • Wechat: fab fa-weixin
  • QQ: fab fa-qq
  • Zhihu: fab fa-zhihu

注意： 本主题中使用的 Font Awesome 版本为 5.11.0

3.13 设置打赏二维码图片

在主题文件的 source/medias/reward 文件中，你可以替换成你的的微信和支付宝的打赏二维码图片。

3.14 设置 Music Player

要支持音乐播放，就必须开启音乐的播放配置和音乐数据的文件。

首先，在你的博客 source 目录下的 _data 目录（没有的话就新建一个）中新建 musics.json 文件，文件内容如下所示：

[{
    "name": "五月雨变奏电音",
    "artist": "AnimeVibe",
    "url": "http://xxx.com/music1.mp3",
    "cover": "http://xxx.com/music-cover1.png"
}, {
    "name": "Take me hand",
    "artist": "DAISHI DANCE,Cecile Corbel",
    "url": "/medias/music/music2.mp3",
    "cover": "/medias/music/cover2.png"
}, {
    "name": "Shape of You",
    "artist": "J.Fla",
    "url": "http://xxx.com/music3.mp3",
    "cover": "http://xxx.com/music-cover3.png"
}]

注：以上 JSON 中的属性：name、artist、url、cover 分别表示音乐的名称、作者、音乐文件地址、音乐封面。

然后，在 主题配置文件中 激活配置即可：

# 是否在首页显示音乐.
music:
  enable: true
  showTitle: false
  title: 听听音乐
  fixed: false # 是否开启吸底模式
  autoplay: false # 是否自动播放
  theme: '#42b983'
  loop: 'all' # 音频循环播放, 可选值: 'all', 'one', 'none'
  order: 'list' # 音频循环顺序, 可选值: 'list', 'random'
  preload: 'auto' # 预加载，可选值: 'none', 'metadata', 'auto'
  volume: 0.7 # 默认音量，请注意播放器会记忆用户设置，用户手动设置音量后默认音量即失效
  listFolded: false # 列表默认折叠
  listMaxHeight: # 列表最大高度

3.15 设置首页/分页文章显示篇数

为了保证文章列表在各个屏幕下都能较好的显示，建议打开 站点配置文件 ，查找 per_page 关键字。修改两个 per_page 的分页条数值均为 6 的倍数，如 12、18 等….，如下：

# Home page setting
# path: Root path for your blogs index page. (default = '')
# per_page: Posts displayed per page. (0 = disable pagination)
# order_by: Posts order. (Order by date descending by default)
index_generator:
  path: ''
  per_page: 6            # per_page
  order_by: -date

# .....
# .....

# Pagination
## Set per_page to 0 to disable pagination
per_page: 12            # per_page
pagination_dir: page

4. Matery 定制

除了上述主题配置，在 Matery【 主题配置文件】中，还可以修改部分自定义信息。

有以下几个部分：

  • 菜单（查找 menu 关键字）
  • 我的梦想（查找 dream 关键字）
  • 首页的音乐播放器和视频播放器配置（查找 music/video 关键字）
  • 是否显示推荐文章名称和按钮配置（查找 recommend 关键字）
  • favicon 和 Logo（查找 favicon/logo 关键字）
  • 个人信息（查找 profile 关键字）
  • TOC 目录（查找 toc 关键字）
  • 文章打赏信息（查找 reward 关键字）
  • 复制文章内容时追加版权信息（查找 copyright 关键字）
  • MathJax（查找 mathjax 关键字）
  • 文章字数统计、阅读时长（查找 postInfo 关键字）
  • 点击页面的’爱心’效果（查找 clicklove 关键字）
  • 我的项目（查找 myProjects 关键字）
  • 我的技能（查找 mySkills 关键字）
  • 我的相册（查找 myGallery 关键字）
  • Gitalk、Gitment、Valine 和 disqus 评论配置（查找 gitalk/gitment/disqus/livere/valine 关键字）
  • 不蒜子统计，谷歌分析（Google Analytics）以及百度分析（查找 busuanziStatistics/googleAnalytics/baiduAnalytics 关键字）
  • 默认特色图的集合：当文章没有设置特色图时，本主题会根据文章标题的 hashcode 值取余，来选择展示对应的特色图。

个人博客应该都有自己的风格和特色。如果本主题中的诸多功能和主题色彩你不满意，可以在主题中进行自定义修改，很多更自由的功能和细节点的修改难以在【主题配置文件】中完成，需要修改源代码才来完成。以下列出了可能对你有用的地方：

4.1 页面配色

关于 Matery 主题页面默认配色方案（包括背景/Banner颜色，顶部导航栏颜色，底部状态栏颜色以及顶部导航栏右上角 Github 图标颜色等）仅代表个人审美。

个人想法，原来主题背景颜色都是偏紫色、橘系的，而顶部导航栏以及底部状态栏颜色是绿色来显示，整体看起来很突兀…..这看起来很不美观。

下面我们来看如何修改主题页面配色：

1 –> 顶部导航栏、底部状态栏、标签页等的背景颜色

可以通过在主题文件的 /source/css/matery.css 文件中，搜索 .bg-color 关键字来进行定制：

/* 顶部导航栏、底部状态栏、标签页等的背景颜色定制 */
.bg-color {
    background-image: linear-gradient(to right, #4cbf30 0%, #0f9d58 100%);
}

2 –> 主题背景颜色动态切换

关于主题背景颜色动态切换修改，可以搜索 webkit-keyframes rainbow & keyframes rainbow 关键字来进行定制：

@-webkit-keyframes rainbow {
   /* 动态切换背景颜色定制（Banner） */
}

@keyframes rainbow {
    /* 动态切换背景颜色定制（Banner） */
}

3 –> 导航栏右上角 Github 图标颜色

上面我们修改了导航栏颜色后，发现导航栏右上角 Github 图标颜色还是绿色状态，故需要特别修改一下。

在 hexo-theme-matery\layout\_partial 下找到 github-link.ejs 文件，搜索 .github-corner svg 关键字，可以查看到内容如下：

.github-corner svg {
    color: #0f9d58;
    fill: #fff;
    height: 64px;
    width: 64px;
}

这里直接修改 color 属性值就可以随心所欲的改变 Github 角标的颜色了~

4.2 背景(Banner)和博文特色图定制

1 –> Matery 主题背景（Banner）图定制

关于 Matery 主题背景（Banner）图，你可以直接在 /source/medias/banner 文件夹中更换你喜欢的 Banner 图片，主题代码中是每天动态切换一张，只需 7 张即可（每天都是新心情~）。

并且，如果你会 JavaScript 代码，可以修改成你自己喜欢切换逻辑，如：随机切换等。

Banner 切换的代码位置在 /layout/_partial/bg-cover-content.ejs 文件的 <script></script> 代码中：

$('.bg-cover').css('background-image', 'url(/medias/banner/' + new Date().getDay() + '.jpg)');

2 –> 博文特色图定制

关于 Hexo 博文封面图，在 /source/medias/featureimages 文件夹中默认有 24 张特色图片，你可以自行增加或者减少。

并且还需要在【主题配置文件】中，搜索 featureImages 关键字进行同步的添加或减少变更：
